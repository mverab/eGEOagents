## AI Transparency

E-GEO is developed with AI assistance. We believe in transparency about what's AI-generated and what's human-authored.

### What's AI-Generated

- **Agent configurations:** The `.claude/agents/` and `.claude/skills/` files are co-written with AI
- **Command implementations:** The `.claude/commands/` files use AI-assisted patterns
- **Documentation:** README and guides are AI-assisted with human review
- **Output templates:** GEO optimization outputs are generated by AI

### What's Human-Authored

- **Project vision and requirements:** The GEO methodology and research grounding
- **Quality control:** All outputs are reviewed for accuracy and brand alignment
- **Community guidelines:** Code of conduct, contributing guidelines
- **Strategic decisions:** Feature prioritization and roadmap

### Our Commitment

- All AI-generated content is reviewed by humans before publication
- We never fabricate data, testimonials, or statistics
- Performance claims cite the E-GEO research paper (arXiv:2511.20867)
- Generated outputs include "AI-assisted" disclosures

### Contributing with AI

We welcome AI-assisted contributions! When submitting PRs:
1. Disclose AI assistance in your PR description
2. Ensure human review of all AI-generated code
3. Follow our coding standards and testing guidelines
4. Be prepared to explain AI-generated code during review

## Getting Started

### Prerequisites
- Python 3.11 or higher
- Git
- A GitHub account

### Setup
1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/mverab/eGEOagents.git
   cd eGEOagents
   ```
3. Create a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```
4. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

## Development Workflow

### Branch Naming
Use descriptive branch names:
- `feature/your-feature-name`
- `bugfix/your-bug-fix`
- `docs/your-doc-update`

### Making Changes
1. Create a new branch from `main`
2. Make your changes
3. Write/update tests
4. Update documentation
5. Commit with clear messages
6. Push to your fork
7. Open a pull request

## Pull Request Process

1. **Before opening a PR:**
   - Check existing PRs to avoid duplicates
   - Ensure your code passes all tests
   - Update documentation as needed

2. **When opening a PR:**
   - Use the PR template
   - Link related issues
   - Describe what changed and why
   - Reference relevant E-GEO paper findings if applicable

3. **After opening:**
   - Respond to review feedback
   - Make requested changes
   - Keep the PR focused and small

## Coding Standards

### Python
- Follow PEP 8
- Use type hints where appropriate
- Write docstrings for functions and classes
- Keep functions focused and small

### Markdown
- Use semantic line breaks
- Include proper header hierarchy
- Add ALT text for images

### GEO-Specific Guidelines
- Cite the E-GEO paper (arXiv:2511.20867) for methodology changes
- Preserve the 10 universal GEO features in optimizations
- Test against multiple AI engines when possible

## Testing Guidelines

### Running Tests
```bash
# Run all tests
pytest

# Run specific test file
pytest tests/test_geo_analyzer.py

# Run with coverage
pytest --cov=egeo --cov-report=html
```

### Writing Tests
- Test both success and failure cases
- Use descriptive test names
- Mock external API calls
- Aim for >80% coverage
